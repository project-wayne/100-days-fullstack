text-align

font-weight
Understanding CSS inheritance through parent elements
Using the <body> selector to apply styles globally
💡 Key Takeaways
Semantic HTML improves structure and readability.
Lists should use <ul> and <li> instead of multiple paragraphs.
Headings should reflect the content hierarchy.
Parent elements can pass certain styles to child elements.
Writing less CSS is often better when inheritance can do the work.
⚠️ Mistakes I Learned From
Repeating text-align: center; across multiple selectors when it could be applied once to the parent.
Placing <header> and <footer> inside <main> without thinking about page structure.
Not considering text contrast against the page background.
🚀 Reflection

I built my first semantic HTML page using <header>, <main>, <section>, and <footer>. I practiced organizing content based on meaning rather than appearance and learned how CSS inheritance can reduce repetitive code. I also started thinking about page structure and accessibility.
